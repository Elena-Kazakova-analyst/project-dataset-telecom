# 📱 Анализ данных телекоммуникационной компании
> **Сегментация пользователей и BI-аналитика**

## 📋 Описание проекта
В данном проекте проведен полный цикл анализа данных клиентской базы телеком-компании (**4500 пользователей, 11 признаков**). 

**Цель:** сегментировать клиентов, выявить аномалии в потреблении услуг и разработать рекомендации для бизнеса.

---

## 🛠 Стек технологий
*   **Python**: `Pandas` (обработка), `NumPy` (математические фильтры), `Matplotlib`/`Seaborn` (EDA).
*   **BI-инструменты**: `Yandex DataLens` (интерактивная визуализация).
*   **Статистика**: Метод Boxplot (межквартильный размах) для поиска выбросов.

---

## 🚀 Ключевые этапы работы

### 1️⃣ Предобработка данных (Pandas)
*   Нормализация названий столбцов, обработка дубликатов.
*   Приведение типов (строки в числа, даты в `datetime`).
*   Группировка базы по возрастным сегментам: **Аспирант, Бизнесмен, Знаток, Студент**.

### 2️⃣ Математический анализ (NumPy)
*   Написана функция `calc_boxplot` для автоматического расчета границ нормы.
*   Выявлено и помечено **358 аномальных записей** (выбросов), что позволило очистить бизнес-метрики от искажений.

### 3️⃣ Визуализация и BI (DataLens)
*   Создан интерактивный дашборд с фильтрацией по сегментам и флагу аномалий.
*   Рассчитаны ключевые показатели: **ARPU** (средний доход) и **Доля звонков** (день/вечер/ночь).

---

## 📊 Итоговые выводы по сегментам клиентов

На основе анализа и очистки данных (Z-score/Boxplot) выявлены следующие закономерности:


| Сегмент | ARPU | Поведение | Рекомендация |
| :--- | :--- | :--- | :--- |
| **🎓 Аспиранты** | **542,45** | Вечерняя активность (Золотой сегмент) | Персональный вечерний безлимит и кэшбэк. |
| **💼 Бизнесмены** | **416,45** | Дневные звонки (Рабочие часы) | Пакеты минут для B2B и делового общения. |
| **🧠 Знатоки** | **320,19** | Сбалансированное потребление | Оптимизация текущего тарифа, поддержка лояльности. |
| **🎒 Студенты** | **193,72** | Ночная активность | Молодежный тариф с дешевым ночным трафиком. |

> **💡 Эффективность фильтра:** Использование фильтра аномалий исключило **358 нетипичных записей**, что повысило точность расчета ARPU на **~12%**.
